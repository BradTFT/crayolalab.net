About

My HomeLab project began before I even knew what a homelab was. When I was younger, I always enjoyed experimenting with various technical items, including laptops, circuits, and anything else i could get my hands on. Later on, I began experimenting with Linux distributions which led me to want to learn even more about programming and how computers work. I had a blast writing code and trying new things (and breaking them) on different peices of hardware. Although, being young, I didnt have the money for new or even used hardware, so I slowly stopped experimenting. A few years later, I received a free HP Z400 Workstation and decided to get back into messing around with things. You can read my full journey later in the page when I go more in depth about the Z400. Shortly after that, I began studying for my Cybersecurity and Network Engineering degree where I was able to learn more about networks and how computers talk to each other. This led me to want to acquire more hardware and learn everything I could about open source tools and hosting my own services. That leads us to today, where my HomeLab is where I have learned quite a bit about troubleshooting, networking, programming, device configuration, and device hardware. If you are considering starting a homelab, take this as a sign to go ahead and start. You dont need to spend a lot, you can even find some stuff for free, but go be curious, and you might just find a new career.

Software

As far as software goes, the setup is pretty simple. All devices are using a compatible version of Ubuntu and Docker. This setup is super simple and allows me to have great control over all services via Docker networking and the nature of containerization.

Resources used

One of the main struggles in this hobby is the lack of support. When I first started, I had no clue how to set things up or fix them when they wouldnt work. Though a large portion of fixing things is just getting in there and trying stuff, some great resources have always been YouTube, app documentation, app github pages, and StackOverflow. For YouTube resources, I highly recommend Hardware Haven, Network Chuck, CrossTalk Solutions, Techno Tim, and FireShip, just to name a few.
